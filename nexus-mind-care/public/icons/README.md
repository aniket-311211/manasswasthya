# PWA Icons for Nexus Mind Care

This directory contains all the required icons for Progressive Web App functionality.

## Required Icon Sizes

You need to create the following icon sizes for full PWA support:

### Standard Icons
- `icon-16x16.png` - 16x16px (favicon)
- `icon-32x32.png` - 32x32px (favicon)
- `icon-72x72.png` - 72x72px (Android)
- `icon-96x96.png` - 96x96px (Android)
- `icon-128x128.png` - 128x128px (Android)
- `icon-144x144.png` - 144x144px (Android)
- `icon-152x152.png` - 152x152px (iOS)
- `icon-192x192.png` - 192x192px (Android, PWA standard)
- `icon-384x384.png` - 384x384px (Android)
- `icon-512x512.png` - 512x512px (PWA standard)

### Shortcut Icons
- `chat-icon-96x96.png` - 96x96px (Chat shortcut)
- `assessment-icon-96x96.png` - 96x96px (Assessment shortcut)
- `resources-icon-96x96.png` - 96x96px (Resources shortcut)

## Icon Design Guidelines

For a mental health app, consider these design principles:

1. **Calming Colors**: Use soft, calming colors like teal, blue, or green
2. **Simple Design**: Keep the design clean and uncluttered
3. **Accessibility**: Ensure good contrast and readability
4. **Consistency**: Use the same design language across all sizes
5. **Professional**: Maintain a professional, trustworthy appearance

## Recommended Tools

- **Figma**: For creating scalable vector icons
- **Canva**: For quick icon generation
- **PWA Builder**: Microsoft's tool for generating PWA assets
- **Favicon.io**: For generating favicon sets

## Mental Health Considerations

- Avoid overly bright or jarring colors
- Use rounded corners for a softer appearance
- Consider using symbols related to wellness (heart, brain, leaf, etc.)
- Ensure the icon conveys trust and safety

## How to Generate Icons

1. Create a master icon at 512x512px
2. Use image editing software to resize to all required sizes
3. Ensure each icon maintains clarity at its target size
4. Test icons on different backgrounds (light/dark)
5. Verify icons look good on both mobile and desktop

## File Naming Convention

All icons should be in PNG format and follow the naming pattern:
- `icon-{size}x{size}.png` for standard icons
- `{feature}-icon-{size}x{size}.png` for feature-specific icons
